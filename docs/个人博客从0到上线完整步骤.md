# 个人博客:从 0 到上线完整步骤

> 技术栈:Spring Boot(Java)+ Vue 3(TypeScript)+ MySQL + Nginx + Docker Compose
> 部署:已有 4核4G 国内云服务器 · 需 ICP 备案
> 用法:按阶段推进,完成一项勾一项。备案那条线可以和开发并行,越早启动越好。

---

## 阶段总览

| 阶段 | 内容 | 大致耗时 |
|---|---|---|
| 0 | 环境准备 | 0.5 天 |
| 1 | 项目初始化(前后端骨架) | 1 天 |
| 2 | 数据库设计 | 0.5 天 |
| 3 | 后端开发 | 1~2 周 |
| 4 | 前端开发 | 1~2 周 |
| 5 | 本地联调 | 几天 |
| ★ | 域名 + 备案(并行) | 2~6 周(行政线) |
| 6 | 服务器环境 + 部署 | 2~3 天 |
| 7 | 上线验证 + 公安备案 | 1~2 天 |
| 8 | 维护 | 持续 |

**关键提醒:** 一读到这份文档,就先去启动「★ 域名 + 备案」。备案是行政流程、卡时间最久,你开发的这几周正好用来等审核,两条线并行能省掉一大段干等的时间。

## 阶段 0:环境准备

本地装好这些:

- **JDK 21**(Spring Boot 3 要求 17+):`java -version` 验证
- **Maven 3.8+**:`mvn -v` 验证(或用 IDEA 自带的)
- **Node.js 18+ / 20+**:`node -v`、`npm -v` 验证
- **MySQL 8**:本地装一个,或用 Docker 跑一个
- **IDE**:后端 IntelliJ IDEA,前端 VS Code
- **Git**:`git --version`,并在 GitHub / Gitee 建好仓库

建议一开始就建两个 Git 仓库:`blog-backend` 和 `blog-frontend`,前后端分离,各自独立提交。(本仓库为my-blog-backend)

**检查点:** 五个命令都能打印出版本号 → 通过。

## 阶段 1:项目初始化

### 后端骨架

用 [start.spring.io](https://start.spring.io) 生成项目,勾选依赖:

- Spring Web
- Spring Data JPA(或你更熟的 MyBatis-Plus)
- MySQL Driver
- Spring Boot DevTools
- Lombok
- Validation

包结构(单体,分层清晰即可):

```
com.yourname.blog
├── controller     // 接口层
├── service        // 业务层
├── mapper / repo  // 数据访问层
├── entity         // 实体
├── dto            // 入参/出参对象
├── config         // 配置(跨域、安全等)
├── common         // 统一返回体、异常处理、工具
└── BlogApplication.java
```

先跑通 `BlogApplication`,浏览器访问 `localhost:8080` 不报错即可。

### 前端骨架

```bash
npm create vite@latest blog-frontend -- --template vue-ts
cd blog-frontend
npm install
npm install vue-router pinia axios element-plus
npm run dev
```

目录建议:

```
src
├── api          // axios 请求封装,按模块分文件
├── router       // 路由
├── store        // pinia 状态
├── views        // 页面级组件
├── components   // 复用组件
├── utils        // 工具(request 拦截器等)
└── assets
```

**检查点:** 后端能启动、前端 `npm run dev` 能打开默认页 → 通过。

## 阶段 2:数据库设计

个人博客最小表集:

- `article`:文章(id、标题、正文、摘要、封面图 URL、状态、阅读量、创建/更新时间)
- `category`:分类
- `tag` + `article_tag`:标签(多对多)
- `comment`:评论(关联文章 id、父评论 id 做楼中楼)
- `user`:后台管理员(就你自己,存用户名 + 加密后的密码)

要点:

- 正文字段用 `LONGTEXT`(支持 Markdown 原文)
- 时间统一用 `datetime`,加 `create_time` / `update_time`
- 加软删除字段 `is_deleted` 而不是物理删除
- 常用查询字段(如 `article.status`、`create_time`)加索引

**检查点:** 在本地 MySQL 建好库和表,能手动插一条测试文章 → 通过。

## 阶段 3:后端开发

按这个顺序写,每写完一个模块就用 Postman / Apifox 测一遍接口:

1. **统一返回体 + 全局异常处理**
   先把 `Result<T>`(code / message / data)和 `@RestControllerAdvice` 全局异常处理写好,后面所有接口都用统一格式返回。这一步先做,能让后面所有接口都干净。

2. **跨域配置(CORS)**
   前后端分离本地开发必然跨域,在 config 里允许前端地址(`localhost:5173`)。

3. **文章模块(核心)**
   - `GET /api/articles`:分页列表(带分类/标签筛选)
   - `GET /api/articles/{id}`:详情(顺便阅读量 +1)
   - `POST /api/articles`:发布(需登录)
   - `PUT /api/articles/{id}`:编辑(需登录)
   - `DELETE /api/articles/{id}`:删除(需登录,软删除)

4. **分类 / 标签模块**:增删改查,前台只读、后台可写。

5. **评论模块**:游客提交评论(建议加审核字段,默认待审核)、列表查询。

6. **鉴权模块(JWT)**
   - `POST /api/login`:校验账号密码,签发 JWT
   - 写一个拦截器 / 过滤器,校验需要登录的接口的 token
   - 密码用 BCrypt 加密存储,**绝不明文**

**检查点:** Postman 里整套接口都能跑通(登录拿 token → 带 token 发文章 → 列表能查到)→ 通过。

---

## 阶段 4:前端开发

1. **封装 axios(`utils/request.ts`)**
   统一 baseURL、请求拦截器自动带上 token、响应拦截器统一处理错误码。这步先做,后面所有请求都走它。

2. **路由 + 布局**
   - 前台:首页(文章列表)、文章详情、分类/标签页、归档
   - 后台:登录页、文章管理、写文章(Markdown 编辑器,可用 `md-editor-v3`)

3. **页面开发**:从首页文章列表 → 详情页 → 后台写文章,逐个对接接口。

4. **响应式适配**
   用 CSS 媒体查询 + 弹性布局,或 Element Plus 的栅格。关键断点:手机(<768px)单列、隐藏侧边栏改抽屉。每写完一个页面,用浏览器开发者工具的手机模拟视图过一遍。

5. **Markdown 渲染**:详情页把正文用 `markdown-it` 渲染成 HTML,记得做 XSS 过滤。

**检查点:** 浏览器和手机模拟视图下,前台能看文章、后台能登录发文章 → 通过。

## 阶段 5:本地联调

- 前端连本地后端,跑通完整链路:登录 → 发文章 → 前台展示 → 评论。
- 处理边界:空列表、长文章、超长标题、特殊字符、未登录访问后台。
- 这一步全部在本地完成,不依赖任何上线手续。

**检查点:** 一条文章从后台发布到前台展示、评论、删除,全流程无报错 → 通过,可以准备上线。

## 阶段 ★:域名 + 备案(尽早并行启动)

> 这条线和阶段 3~5 同时进行。

1. **注册域名**:阿里云万网买 `.com` 或 `.cn`,完成域名实名认证(实名信息要和备案主体一致,都是你本人)。
2. **提交 ICP 备案**:在服务器所在云厂商的备案系统提交。需要身份证、手机活体刷脸、网站名称(取朴素名,别含"中国/新闻/博客网"等敏感词)、网站类型选"个人博客/空间"。
3. **等审核**:云厂商初审 → 省通信管理局终审,整体约 1~4 周。期间你继续开发。
4. **拿到备案号**:形如 `京ICP备xxxxxxx号`,上线后必须挂在网站页脚并链接到工信部网站。

**检查点:** 收到备案通过短信、拿到备案号 → 才能正式对外开放域名访问。

## 阶段 6:服务器环境 + 部署

备案通过(或先用 IP 内部测试)后:

### 6.1 服务器装环境

SSH 登录你的 4核4G 服务器,安装 Docker 和 Docker Compose:

```bash
# 安装 Docker
curl -fsSL https://get.docker.com | bash
systemctl enable --now docker
# 验证
docker -v && docker compose version
```

安全组 / 防火墙:只开放 `22`(SSH)、`80`、`443`,数据库端口 `3306` 不要对外开放。

### 6.2 准备部署文件

在服务器建一个目录,例如 `/opt/blog`,放:

```
/opt/blog
├── docker-compose.yml
├── nginx/
│   ├── nginx.conf
│   └── cert/            // SSL 证书放这
├── backend/
│   └── blog.jar         // 后端打包产物
└── frontend/
    └── dist/            // 前端打包产物
```

- 后端打包:本地 `mvn clean package -DskipTests` → 得到 `blog.jar`
- 前端打包:本地 `npm run build` → 得到 `dist/`
- 用 `scp` 或 Git 把产物传到服务器

### 6.3 docker-compose.yml 骨架

```yaml
services:
  mysql:
    image: mysql:8
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: 你的强密码
      MYSQL_DATABASE: blog
    volumes:
      - ./mysql-data:/var/lib/mysql
    # 不映射端口到宿主机外网,只在容器网络内访问

  backend:
    image: eclipse-temurin:17-jre
    restart: always
    working_dir: /app
    volumes:
      - ./backend:/app
    command: java -jar blog.jar
    depends_on:
      - mysql

  nginx:
    image: nginx:alpine
    restart: always
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf
      - ./nginx/cert:/etc/nginx/cert
      - ./frontend/dist:/usr/share/nginx/html
    depends_on:
      - backend
```

> 注意:后端配置文件里数据库地址要写成 `mysql`(容器服务名),不是 `localhost`。

启动:

```bash
cd /opt/blog
docker compose up -d
docker compose logs -f   # 看启动日志
```

**检查点:** 三个容器都是 running,日志无报错 → 通过。

## 阶段 6.5:Nginx + HTTPS 配置

### 申请 SSL 证书

云厂商控制台申请免费 SSL 证书(绑定你的域名),下载 Nginx 格式的证书文件,放到 `/opt/blog/nginx/cert/`。

### nginx.conf 核心规则

```nginx
server {
    listen 80;
    server_name yourdomain.com;
    return 301 https://$host$request_uri;   # 强制跳 HTTPS
}

server {
    listen 443 ssl;
    server_name yourdomain.com;

    ssl_certificate     /etc/nginx/cert/your.pem;
    ssl_certificate_key /etc/nginx/cert/your.key;

    # 前端静态资源
    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;   # 支持 Vue history 路由
    }

    # 后端 API 反向代理
    location /api/ {
        proxy_pass http://backend:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

改完 `docker compose restart nginx`。

### 域名解析

在云厂商把域名 A 记录指向服务器公网 IP。等解析生效(几分钟到几十分钟)。

**检查点:** 浏览器访问 `https://yourdomain.com` 能打开博客、地址栏是小锁、`/api` 接口正常 → 通过。

## 阶段 7:上线验证 + 公安备案

1. **全面验证**:用电脑和手机分别访问,过一遍登录、发文、评论、各页面响应式。
2. **挂备案号**:页脚加上 ICP 备案号,并链接到 `https://beian.miit.gov.cn`。
3. **公安备案**:上线后 30 天内,在「全国互联网安全管理服务平台」提交公安备案,通过后把公安备案号也挂到页脚。
4. **(可选)SEO**:在百度站长平台提交站点;若用的是 SPA 收录不佳,考虑后续上 SSR。

**检查点:** 域名可公开访问、双备案号已展示 → 正式上线完成。

## 阶段 8:维护

- **数据库备份**:写个定时脚本 `mysqldump` 导出,定期备份(别等出事才想起)。
- **证书续期**:免费 SSL 证书通常一年期,到期前续期并替换。
- **日志与监控**:定期看容器日志、服务器资源占用。
- **安全**:只开必要端口、SSH 用密钥登录、及时更新依赖。
- **迭代**:有了稳定底座,再慢慢加功能(全文搜索、阅读量统计、RSS、评论审核后台等)。

## 给求职加分的几个"设计决策"(面试时主动讲)

做的时候留意这几点,面试讲项目时这些比功能本身更能体现你的思考:

- **为什么用单体而非微服务**:个人博客流量小、单人维护,微服务的拆分成本和运维复杂度远超收益——能讲清"在什么场景下才该上微服务",反而显出你懂架构取舍。
- **JWT 无状态鉴权**:为什么用 JWT 而不是 Session、token 怎么签发和校验、怎么防止泄露。
- **图片为什么走 OSS**:本地磁盘的弊端(占空间、难迁移、难加 CDN),对象存储如何解决。
- **统一返回体 + 全局异常处理**:如何让前端拿到一致的错误结构,提升前后端协作效率。
- **前后端分离的部署形态**:Nginx 如何同时托管静态资源和反向代理 API。
