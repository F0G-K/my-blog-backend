开发个人博客这事你之前做过,前后端分离主要是把"展示层"和"数据/逻辑层"彻底拆开,前端是独立的 SPA,后端只提供 JSON 接口,两者通过 HTTP 通信。先给你看整体架构,再讲流程。你之前做过博客,所以这里我重点讲跟"前后端分离"相关的部分,普通的功能开发就略过。整个流程大致是这样:

**1. 需求与功能拆解**
先把功能列死,别边做边改。个人博客核心一般是:文章 CRUD、分类/标签、Markdown 编辑与渲染、首页列表+详情页、归档、评论,以及一个管理员后台(至少要能登录后写文章)。再看要不要加搜索(你有 ES 基础,可以用 Elasticsearch 做全文检索,也是个可以在面试里讲的点)。

**2. 技术选型(按你熟的栈)**
- 后端:Spring Boot + MyBatis-Plus(或 JPA)+ MySQL,鉴权用 JWT,可加 Redis 缓存热点文章/做 token 黑名单。
- 前端:Vue3 + Vite + Vue Router + Pinia + Axios,UI 用 Element Plus 或 Naive UI,Markdown 编辑器用 `md-editor-v3` 之类的现成组件。
- 文档:Knife4j / Swagger 自动生成接口文档。

**3. 数据库设计**
先画 ER 图再建表。核心表:`article`、`category`、`tag`、`article_tag`(多对多中间表)、`comment`、`user`。这步定下来,后端实体和接口才有依据。

**4. 接口设计——这是前后端分离的关键**
分离架构的核心不是"用了 Vue",而是先把 API 契约定清楚:每个接口的 URL、请求方法、入参结构、返回结构、状态码,以及一个统一返回格式(比如 `{ code, message, data }`)。契约定好之后,前端可以拿 Mock 数据先写页面,后端同时写实现,两边并行,不用互相等。这也是图里中间那条边界的意义。

**5. 后端开发**
按 Controller → Service → Mapper 分层写。前后端分离里几个必做项:统一返回对象、全局异常处理(`@RestControllerAdvice`)、跨域配置(CORS,因为前后端不同源)、JWT 登录鉴权(拦截器/过滤器校验 token)、参数校验(`@Valid`)。

**6. 前端开发**
搭好 Vite 脚手架,配路由,**封装 Axios**(请求拦截器统一加 token,响应拦截器统一处理错误和 401 跳登录),然后按页面拆组件,对着接口文档调用。

**7. 前后端联调**
把 Mock 换成真实接口,解决跨域、token 过期、字段对不上这类问题。

**8. 部署上线**
前端 `npm run build` 打成静态文件,用 Nginx 托管,同时由 Nginx 反向代理 `/api` 到后端;后端打 jar 包跑在服务器上(或者用 Docker 容器化,你有 Docker 基础);配好 MySQL,有域名再上 HTTPS。

一句话抓重点:**前后端分离的灵魂是"接口契约先行"**,定好契约后两端独立开发、独立部署,靠 HTTP+JSON 通信。
