# 个人博客 · 后端开发指导文档(v1.0)

> 配套:《需求与技术方案 v1.0》《接口契约 v1.1》《建表脚本》《docker-compose》。本文是后端开发者的上手指南,照着即可从零搭起。

---

## 一、目标概述

前后端分离的个人博客**后端**,只提供 JSON 接口。分两类使用者:

- **游客**:浏览已发布文章、分类 / 标签、归档、搜索、提交评论(进审核)。
- **管理员(唯一)**:登录后写作 / 发布 / 编辑 / 删除文章、审核评论、管理开源项目与站点信息。

接口分**公开**(无需鉴权)和**管理**(JWT 鉴权)两类,统一前缀 `/api`。

### 2.1 核心依赖与坑点(⚠️ 别踩)

- `mybatis-plus-spring-boot3-starter:3.5.15` —— **必须是 boot3 版**,老的 `mybatis-plus-boot-starter` 是 SB2 的,会因 javax/jakarta 报错。
- `jjwt-api / jjwt-impl / jjwt-jackson:0.12.6` —— **0.12.x 的 API 和老教程差异大**,签发 / 解析写法变了,别照抄旧博客。
- `knife4j-openapi3-jakarta-spring-boot-starter:4.5.0` —— **必须 jakarta 版**,用错包启动报错。
- `io.minio:minio:9.0.2`、`spring-boot-starter-data-redis`、`spring-boot-starter-data-elasticsearch`、`spring-boot-starter-security`、`spring-boot-starter-validation`、`spring-boot-starter-web`。

## 三、工程结构(单体即可)

```
src/main/java/com/xxx/blog/
├─ controller/    # 接口层,参数校验
├─ service/       # 业务层,事务,ES/Redis 协作
│  └─ impl/
├─ mapper/        # MyBatis-Plus 数据访问
├─ entity/        # 数据库实体

```

## 四、环境准备

1. 装 JDK 21、Maven。
2. 起中间件(见《docker-compose》)。**Ubuntu 先跑** `sudo sysctl -w vm.max_map_count=262144`,再 `docker compose up -d`。
3. 建库:执行《建表脚本》(8 张表 + 初始化),**记得把 admin 的 `password_hash` 占位串换成真实 BCrypt 哈希**。
4. 配 `application.yml`(本地连 localhost,密码统一 你的密码(自行设置)):

```yaml
server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/blog?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: root
    password: 你的密码(自行设置)
  data:
    redis:
      host: localhost
      port: 6379
      password: 你的密码(自行设置)
  elasticsearch:
    uris: http://localhost:9200
    username: elastic
    password: 你的密码(自行设置)

mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true     # 下划线↔驼峰
  global-config:
    db-config:
      logic-delete-field: is_deleted        # 逻辑删除(article)
      logic-delete-value: 1
      logic-not-delete-value: 0

minio:
  endpoint: http://localhost:9000
  access-key: minioadmin
  secret-key: 你的密码(自行设置)
  bucket: blog

jwt:
  secret: 换成一段足够长的随机密钥不要外泄
  expire-minutes: 1440
```

## 五、数据库要点

8 张表:`admin`、`site_info`、`category`、`tag`、`article`、`article_tag`、`comment`、`project`(字段见建表脚本)。设计约定:

- **逻辑外键**:不建物理外键(阿里规范),约束在应用层;关联列(`category_id`、`article_id`、`tag_id`)都建了索引。
- **状态用 TINYINT**:`article.status` 0 草稿 / 1 已发布;`comment.status` 0 待审 / 1 通过 / 2 拒绝。接口对外仍用字符串,**由后端做枚举映射**。
- **逻辑删除**:`article.is_deleted`(配合 MyBatis-Plus `@TableLogic`),查询自动过滤。
- **索引**:文章列表走 `(status, published_at)`;评论走 `(article_id, status)`;标签关联用联合主键 `(article_id, tag_id)`。

## 六、接口规范(契约要点)

- **统一返回**:`{ code, message, data }`,`code=0` 成功。用一个 `Result<T>` 包装,配 `@RestControllerAdvice` 全局异常处理。
- **分页**:入参 `page/pageSize`,返回 `{ list, total, page, pageSize }`(对接 MyBatis-Plus 的 `IPage`)。
- **鉴权**:管理接口校验请求头 `Authorization: Bearer <token>`,失败返回 401。
- **状态枚举映射**:

| 含义 | API 字符串 | DB(TINYINT) |
|---|---|---|
| 文章 草稿 / 已发布 | draft / published | 0 / 1 |
| 评论 待审 / 通过 / 拒绝 | pending / approved / rejected | 0 / 1 / 2 |

> 每个接口的 URL / 入参 / 返回字段以《接口契约文档 v1.1》为准。

## 七、关键模块实现要点

### 7.1 鉴权(JWT + Spring Security)
- 登录:校验 username + BCrypt 密码 → 用 jjwt 0.12.6 签发 JWT 返回。
- 管理接口:过滤器 / Spring Security 校验 token;公开接口放行。
- 单管理员,不开放注册;密码 BCrypt 存储。

### 7.2 文章
- 列表 / 详情对外返回 `categoryName` 与 `tags[]`(JOIN category + article_tag + tag 得到)。
- `cover_url` 是否为空决定前端卡片形态(封面 / 文字)。
- 发布:`status=1` 并写 `published_at`,**同步写 ES**;下架 / 删除**同步从 ES 移除**。
- **草稿自动保存**:`POST /api/admin/articles/autosave`,无 id 新建草稿并返回 id。

### 7.3 评论
- **二级楼中楼**:`parent_id` 为空=顶级,有值=挂在该顶级下的回复;不记录被回复对象。返回时顶级评论嵌套 `replies`。
- **审核**:提交即 `pending`,前台只查 `approved`。
- **安全**:内容做 XSS 净化;提交接口加长度校验 + 同 IP 限流防刷。

### 7.4 搜索(Elasticsearch)
- 索引字段:`title + content + tags`。
- **实时同步**:文章发布 / 编辑 / 删除时更新索引。个人项目可在 Service 里直调;想练解耦可用 Spring 事件(`ApplicationEvent`)异步监听。
- 检索返回命中文章 + 高亮片段。

### 7.5 阅读量防刷(Redis)
- key = `访客标识(IP/cookie) + 文章ID`,写入并设过期(如 1h);命中不计数,未命中才 `view_count + 1`(可异步落库)。

### 7.6 图片上传(MinIO)
- `POST /api/admin/upload`(multipart),转存到 `blog` 桶,返回可访问 URL(桶已设公开读)。限制文件类型与大小(如 ≤5MB)。

### 7.7 站点信息
- `site_info` 单行(id=1),`GET /api/site/info`(含 COUNT 出来的文章 / 标签 / 分类数)对外;管理端 `GET/PUT /api/admin/site-info` 编辑。

### 7.8 公共能力
统一返回 `Result`、全局异常处理、`@Valid` 参数校验、CORS **白名单**(只放行前端域名,别全开)、Knife4j 在线接口文档。

## 八、安全清单

- 密码 BCrypt;JWT 设过期时间。
- 全部入参 `@Valid` 校验;MyBatis-Plus 参数化查询防注入。
- UGC(评论 / 正文)渲染前 XSS 净化。
- CORS 白名单;评论 / 登录接口限流。
- 生产环境中间件端口绑 `127.0.0.1`、密码进 `.env`、安全组只开 80/443。

## 九、建议开发顺序

1. 搭框架 + 配置 + 统一返回 / 全局异常。
2. 实体 + Mapper(MyBatis-Plus 代码生成)。
3. 登录鉴权(JWT)跑通。
4. 文章模块(CRUD + 发布 + 草稿)。
5. 分类 / 标签 / 站点信息 / 开源项目。
6. 评论(提交 + 审核 + 楼中楼)。
7. 图片上传(MinIO)。
8. 搜索(ES 索引 + 同步 + 高亮)。
9. 阅读量防刷、缓存、限流等优化。

## 十、部署

- 后端打 jar,`java -jar` 直跑或容器化进同一 compose(容器内连服务名 `mysql/redis/elasticsearch/minio`)。
- Nginx 托管前端 + 反代 `/api` 到 8080。
- 同机部署:中间件端口**别暴露公网**,绑 `127.0.0.1`;安全组只放行 80/443 + SSH。
